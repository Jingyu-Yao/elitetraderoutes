About

The first version of this website will be focused on providing a restful api. The api will have a matching html and api view in most cases.

To specify the return type of the content you can either use the appropriate accept header (recommended) or append '.json', '.html' or '.api' to the end of your calls. ex. /systems/1/.json will return the data for that endpoint in json and /systems/1/.api will display api information for that endpoint.

The use of accept header is recommended because if a suffix is used, all of the hyperlinks returned on that page will also have the suffix. This can become somewhat bothersome as it makes the return value not very consistent.

The backend used to build this api are Django and django-rest-framework.

The data used in this website is collected from eddb.io and EDDN.
